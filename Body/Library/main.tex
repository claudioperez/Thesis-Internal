

\subsection{Introduction}

\paragraph{Motivation}
The modern landscape of computational mechanics is characterized by trends like (1) the demand for parametric/sensitivity analysis and optimization, 
and (2) advances in LLM technology that facilitate model building/definition.
These trends favor small element libraries that are valid for a wide range structures. 
Furthermore, it is important that salient modeling features are semantically related to the model definition data. eg, it is not good to separate features like warping or shear deformability  into separate elements. 
Rather, there should be only a minimal number of elements, which expose these features as toggleable options. 


This chapter introduces a unified 3D frame element library that builds on the prior developments. 
The work builds on the modular architecture that has been pioneered in work like \cite{filippou1996fedeas}, \cite{desouza2000forcebased}, \cite{scott2008softwarea}. 

Two elements are presented, which are supported by three cross sections and four geometric transformations. 
All cross section models support shear deformations and at least one
warping mode, and work seamlessly with standard frame elements whether or not they take such deformations into account.

Mixed finite element formulations tend to be desirable when inelastic response must be represented, and geometric nonlinearities are expected to be moderate \cite{neuenhofer1997evaluation}. 
Frame elements of this lineage were pioneered by \cite{kaba1984refined}, \cite{ciampi}, \cite{filippou}, \cite{hjelmstad2003mixed}, and many others. 
The development and implementation of these methods tends to be more involved than displacement formulations, and they are typically applied to linearized governing equations. 
  Moderate geometric nonlinearities were considered in 2D by \textcite{neuenhofer1998geometrically} who introduced the concept of \emph{curvature-based displacement interpolation} (CBID), and applied it to the truncated 2D equilibrium equations which govern Euler buckling. 
  This was later extended to the 3D \gls{frame:axial}-\gls{frame:euler} model (Box~\ref{box:axial-euler}) by \cite{desouza2000forcebased}.

\paragraph{Scope}
Only shear-deformable formulations are discussed. 
Shear-deformable beam theories are known to converge to Euler-Bernoullii beam theory in the limit of increasing span-to-depth ratio, and therefore are valid over a larger range of structure types, making them more suitable for the demands of our context.

For similar reasons, only distributed inelasticity formulations are considered (as opposed to concentrated inelasticity formulations). 
Concentrated inelasticity formulations inherently carry several restrictive assumptions pertaining to things like element loading (typically assume no distributed loads) and resultant coupling. 
Furthermore, concentrated inelasticity elements require non-semantic information (need a better way to describe this), like moment-rotation curves. Distributed inelasticity elements, on the other hand, generally only require standard material parameters like the yield stress and Young's modulus.

\paragraph{1. PrismFrame}
The \texttt{Prism} element is a standard linearly elastic prismatic
  frame. This element is ideal for modeling simple slender members that
  are not expected to exhibit inelastic response or experience
  excessively large deformations. 
  Nonlinear effects from moderately
  large displacements can be accounted for using the \texttt{PDelta} or
  \texttt{Corotational} geometric transformations. Additionally, the
  \texttt{kinematics} option can be used to incorporate \(P-\delta\)
  effects with \texttt{Axial}  (\(P-\delta\)) element kinematics. 
  This model is inherently elastic. 
  If an inelastic section is supplied through the \texttt{section} argument,
  resultant stresses and stiffness are obtained by linearizing about the
  strain-free state. 
  The implementation of 3D \gls{frame:axial} element
  kinematics is novel and supports asymmetric sections.

\paragraph{2. ForceFrame}
  The \ForceFrame{} element is ideal for modeling inelasticity in
  frames \citep{neuenhofer1997evaluation}. 
  % Both \gls{frame:shear} and \gls{frame:euler} formulations are
  % supported with \gls{frame:basic} and \gls{frame:axial} (\(P-\delta\)) element kinematics. 
  
  This element is adapted from the \texttt{forceBeamColumn} and
  \texttt{forceBeamColumnCBDI} elements of \OpenSees{} and
  \texttt{Dinel3dFrm\_EBwFF} from FEDEASLab. 


  Implementation details are discussed in \cref{sec:ForceFrame}.


\paragraph{3. CubicFrame}
  The \texttt{Cubic} element fits displacements to a cubic polynomial
  inside a 2-node basic system with optional shearing deformations. This
  element requires internal iterations in the presence of inelastic
  shear deformations. 
  The implementation of this element derives from
  the \texttt{Dinel3dFrm\_EBwDF} and \texttt{Dinel3dFrm\_TwDF*} elements
  of FEDEASLab \citep{filippou1996fedeas}.


\paragraph{4. ExactFrame}
  The \ExactFrame{} element is a \(C^0\) displacement formulation of
  the exact shear-deformable element kinematics. 
  This element is ideal
  for modeling large elastic deformations. 
  Reduced integration must be
  used to avoid shear locking, and consequently a combination of \(h\)
  and \(p\) refinement is required in order to adequately capture
  inelasticity.

Elements are grouped into three types based on their geometric nature:

\begin{itemize}
\tightlist
\item
  \textbf{Type I} elements are formulated in a basic system and can only
  be connected with 2 nodes. 
  Valid geometric transformations for these
  elements include \texttt{Linear}, \texttt{PDelta}, and
  \texttt{Corotational}.
\item
  \textbf{Type II} elements are formulated in a local system and can be
  connected with an arbitrary number of nodes. Valid geometric
  transformations for these elements include \texttt{Linear} and
  \texttt{Corotational}.
\item
  \textbf{Type III} elements are formulated in the global system using
  exact kinematics.
\end{itemize}

\begin{enumerate}
    \item[T1.]\label{lib:T1} \texttt{Linear}
    \item[T2.]\label{lib:T2} \texttt{PDelta}

\end{enumerate}

\subsection{Summary}



This library addresses the following shortcomings in the existing
\OpenSees{} offerings:

\begin{itemize}
\tightlist
\item
  No geometrically exact elements
\item
  Current elements do not handle distributed mass consistently
\item
  Cross section library is disjoint
\item
  Higher-order section kinematics are not handled consistently (e.g.
  Wagner strains)
\item
  Inconsistent treatment of asymmetric sections
\item
  Geometric transformations do not support shear or multi-node frame
  elements
\item
  Idiosyncrasies and inefficiencies in the corotational transformation
\end{itemize}
